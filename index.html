Git ve Github'ın önemi

  1. Sürüm Kontrolü (Version Control): Kodun geçmişteki tüm değişikliklerini kaydeder, böylece istenildiğinde eski sürümlere geri dönülebilir.
  2. Takım Çalışması Kolaylığı: Birden fazla kişi aynı proje üzerinde rahatça çalışabilir, değişiklikler birleştirilebilir.
  3. Kod Yedekleme: GitHub, projeleri bulutta sakladığı için bilgisayar bozulsa bile kod kaybolmaz.
  4. Açık Kaynak Katkısı: GitHub, açık kaynak projelere katkı yapmayı kolaylaştırır, böylece yazılımcılar global toplulukla iş birliği yapabilir.
  5. Kod İncelemesi (Code Review): Pull request’ler sayesinde ekip arkadaşları kodu gözden geçirip yorum yapabilir.
  6. Hata Takibi (Issue Tracking): GitHub projelerdeki bug’ları ve yapılacak görevleri organize etmeyi sağlar.
  7. Dokümantasyon Kolaylığı: README ve Wiki bölümleri sayesinde projeye ait dökümanlar kolayca paylaşılır.
  8. Sürekli Entegrasyon ve Dağıtım (CI/CD): GitHub Actions gibi araçlarla otomatik test, build ve deploy süreçleri yapılabilir.
  9. Şeffaflık ve Portföy: Geliştiriciler GitHub hesaplarını bir portföy gibi kullanarak iş başvurularında çalışmalarını gösterebilir.
  10. Farklı Versiyonları Yönetme (Branching): Yeni özellikler veya deneysel çalışmalar için dallar (branch) açılabilir, ana kod zarar görmez.
  11. Versiyon Kontrolü: Yapılan her değişiklik kaydedilir, geçmişe kolayca dönülebilir.
  12. Takım Çalışması: Birden fazla kişi aynı proje üzerinde sorunsuz çalışabilir.
  13. Branch Kullanımı: Farklı özellikler paralel olarak geliştirilebilir.
  14. Kod Güvenliği: Hatalar geri alınabilir, veri kaybı engellenir.
  15. Bulut Depolama: GitHub sayesinde projeler her yerden erişilebilir hale gelir.
  16. Kod İnceleme: Pull Request ve yorumlama özellikleriyle kalite artar.
  17. Açık Kaynak Ekosistemi: Dünyadaki projelere katkı sağlama ve öğrenme imkanı verir.
  18. Profesyonel Portföy: GitHub profili iş görüşmelerinde güçlü bir referanstır.
  19. Üretkenlik: Kod çatışmalarını azaltır, zamandan tasarruf sağlar.
  20. Kariyer Avantajı: Git ve GitHub bilgisi yazılım sektöründe temel yetkinliktir.
  21. 1. Git nedir? Versiyon kontrol sistemidir.
  22. 2. Yerel çalışır: İnternet olmadan da kullanılabilir.
  23. 3. Commit yapısı: Değişiklikler commit ile kaydedilir.
  24. 4. Branch kullanımı: Farklı özellikler için dallar açılır.
  25. 5. Merge ve Conflict: Branch'ler birleştirilirken çakışmalar olabilir.
  26. 6. GitHub nedir? Git projelerini barındıran platformdur.
  27. 7. Remote Repository: Projenin uzaktaki kopyasıdır.
  28. 8. Pull Request: Değişiklik teklifleri için kullanılır.
  29. 9. Issue sistemi: Hata ve görev takibi yapılır.
  30. 10. Açık kaynak katkısı: Başka projelere katkı sağlanabilir.
